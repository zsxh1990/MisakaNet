## 卡片 1：备份中提取 Payload（负载）配置

### 问题

拿到 FANUC 机器人备份目录，想提取负载配置（质量、重心、惯量），但文件是二进制格式，打不开。

### 文件位置

```
<backup_dir>/SYSVARS.SV     → 用 kconvars 转换
<backup_dir>/CBPARAM.VR     → 用 kconvars 转换
```

### 关键参数

| 参数 | 说明 |
|---|---|
| `$PLST_GRP1[1].$PAYLOAD_MASS` | 负载质量 (kg) |
| `$PLST_GRP1[1].$PAYLOAD_COG_X/Y/Z` | 重心坐标 (mm) |
| `$PLST_GRP1[1].$PAYLOAD_INERTIA_IX/IY/IZ` | 惯量 (kg·m²) |
| `PAYLOAD1` / `PAYLOAD2` (CBPARAM.VR) | MH 夹爪向导负载 |

### 工具

```bash
kconvars.exe SYSVARS.SV > sysvars_decoded.txt
kconvars.exe CBPARAM.VR > cbparam_decoded.txt
```

### 注意

- kconvars 需要 `robot.ini`，非 ASCII 路径会崩溃
- `.vr` 文件头 `FE EF`，`.sv` 文件头 `*SYSTEM*`
- Python `struct.unpack('>f', ...)` 不能直接解 FANUC 二进制

## 卡片 2：最大磨损量查找方法（Spot Weld）

### 问题

需要查找点焊电极头的最大磨损量，用于判断是否需要更换电极。

### 文件位置

```
<backup_dir>/sysspot.sv → 用 kconvars 转换
```

### 关键参数

```
$SPOTEQSETUP[1].$EPAF_TRGDST = 5.00mm  ← 最大磨损量（可动侧电极）
$SPOTEQSETUP[2-5].$EPAF_TRGDST = 5.00mm  ← 其他设备
```

### 关联参数

| 参数 | 说明 |
|---|---|
| `$SPOTEQSETUP[x].$EPAF_TRGDST` | 最大磨损量 (mm) |
| `$SPOTEQSETUP[x].$TD_LIMIT` | 修磨次数限制 |
| `$SPOTCONFIG.$TD_AFTER_CC` | 焊后修磨开关 |
| `R[32]` | 原始电极磨损 |
| `R[33]` | 当前电极磨损 |
| `R[36]` | 修磨磨损量 |

### 注意

- 只有最大磨损量能在备份里找到
- 其他检测参数（负载裕度、电极速度、压力标定）在焊机控制器里，不在机器人备份
- R[32]/R[33]/R[36] 是运行时值，不在备份中
