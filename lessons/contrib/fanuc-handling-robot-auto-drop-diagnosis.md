## FANUC 搬运机器人掉自动模式诊断

### 问题描述

FANUC 搬运机器人负责搬运车顶零件。零件到位后，销式快换耦合器（pin-type quick-change coupler）断开；点焊完成后重新连接。在此过程中，机器人**偶尔掉出自动模式**，需要手动按下电气控制面板上的复位和启动按钮才能恢复。

- **频率**：约每天一次，有时数天不出现
- **影响**：产线停机，需人工干预恢复
- **场景**：搬运→焊接→放件的断连循环中

### 根因分析

所有社区回复指向同一根因：**快换耦合器电气触点的间歇性接触故障**。

```
正常流程：
  抓件 → 快换断开 → 焊接 → 快换重连 → 放件
                    ↑
              信号瞬间丢失 → 控制器退出 AUTO
```

| 根因 | 机制 | 检查方法 |
|------|------|----------|
| 触点磨损/污染 | 断连时接触面氧化或异物导致瞬断 | 目视检查触点表面 |
| 机械对位偏差 | 耦合器对接不精确，连接不稳固 | 检查定位销和导向机构 |
| 信号线虚焊 | 焊接点在振动中断开 | 万用表逐线通断测试 |
| 线束老化 | 线缆内部断丝，弯折时接触不良 | 晃动线束观察信号变化 |

### 修复方法/技术要点

#### 方案一：清洁与检查（优先）

1. 拆卸快换耦合器，检查所有电气触点
2. 用酒精清洁触点表面氧化层和油污
3. 检查触点弹簧压力是否足够
4. 验证耦合器机械对位精度

#### 方案二：信号监控定位

编写一个监控程序，实时记录每个信号线的状态：

```
-- 伪代码：信号监控程序
-- 在后台循环运行，记录信号状态变化
WHILE TRUE DO
  IF DI[n] = OFF AND DI[n]_PREV = ON THEN
    -- 记录信号瞬断时间戳和信号编号
    R[n] = R[n] + 1  -- 累计瞬断次数
  ENDIF
  DI[n]_PREV = DI[n]
  WAIT 0.1 sec
ENDWHILE
```

通过监控程序可以精确定位哪根信号线存在瞬断问题。

#### 方案三：更换线束

若无法通过监控定位具体故障点，最直接的方案是**更换整条线束**。

### 验证方式

1. 清洁/更换触点后连续运行 **3 天以上**无掉自动现象
2. 监控程序记录的信号瞬断次数降为 **0**
3. 更换线束后问题彻底消失
4. 长期观察：每周检查一次耦合器触点状态
